Codex 생산성을 높이는 12가지 엔지니어링 트릭

6월 22, 2026 · 5 분 · Junho Lee
목차

  • 원격 전환과 에이전트 칸반

숨겨진 고수들은 Codex에 12가지 트릭을 더합니다 Codex Desktop App을 설치하고 ChatGPT를 구독했다면 반은 했습니다. 나머지 반은 Codex를 어떤 작업 시스템으로 엮느냐에 달려 있습니다. 제가 사용하는 12가지 트릭을 공유해드립니다. 1/ Codex 크레딧 두 배로 만들기 (Turn ChatGPT into MCP) ☑ ChatGPT는 기획, Codex는 실행으로 구분합니다 ↳ devspace 또는 codexpro 로 ChatGPT가 로컬 repo를 읽고 계획을 세우게 만듭니다 ↳ ChatGPT와 Codex를 같은 작업 공간 위에서 연결하면서 동시에 별도의 ChatGPT 크레딧과 pro 모델을 활용할 수 있습니다 DevSpace: CodexPro: 2/ 계정은 나누고 환경은 공유하기 (Switch Codex Accounts Fast) ☑ 여러 계정을 쉽게 연동해 quota 와 작업 환경을 분리합니다 ↳ CLIProxyAPI 로 provider 와 계정 풀을 API layer 뒤에 연결 ↳ 인증된 여러 개의 계정을 라운드 로빈 방식으로 연결하여 하나의 API 게이트웨이로 사용하는 방식입니다 Codex Advanced Config: codex-auth: CLIProxyAPI: 3/ 이미지 생성을 위한 무한 캔버스 (Open Excalidraw in Browser) ☑ 이미지 수정 작업 중 의도를 말이 아니라 화살표 등 그림으로 전달합니다 ↳ 결과물을 인앱 브라우저에 띄운 Excalidraw 위에 올리고 여기에서 화이트보드 작업을 합니다 ↳ 텍스트 100줄보다 화살표 하나가 빠를 때가 많습니다 Codex Browser: Appshots: Excalidraw 참고: 4/ 이미지 팩토리 병렬 가동하기 (Spawn Imagegen Subagents) ☑ 여러 이미지를 순서대로 하나씩 만드는 것보다 병렬로 한 번에 생성합니다 ↳ imagegen subagent 를 여러 개 띄웁니다 ↳ 밤새 돌아가는 콘텐츠 팩토리 구축에는 속도가 매우 중요합니다 5/ 노트북에서 서버로 작업 넘기기 (Hand Off Local to Remote) ☑ 노트북을 닫기 전 텍스트 요청만으로도 원격 장치에 바로 작업을 이어갈 수 있습니다 ↳ VPS, Mac mini, always-on machine 이 있으면 장시간 작업에 유리합니다 ↳ 노트북 등 local 에서 시작하고 remote 에서 계속 돌리는 방식입니다 Remote Connections: iPhone 원격 사용 참고: 6/ 새롭게 등장한 플러그인 활용하기 (Use Product Design and Record Replay Plugins) ☑ Product Design Plugin으로 프론트 결과물의 완성도를 끌어올립니다 ↳ 현재 Codex 가 자주 놓치는 UI 밀도, 배치, 시각적 완성도를 보완합니다 ☑ Record and Replay Plugin 으로 반복 작업을 녹화해 재사용합니다 ↳ 특정한 나의 작업 방식을 화면으로 보여주어 자동화시키는 것이 핵심입니다 7/ 질문으로 스펙을 완성하기 (Grill Me into a PRD) ☑ 바로 구현하지 않고 먼저 질문으로 요구사항을 고정합니다 ↳ grill-me 또는 grill-with-docs 로 인터뷰하고 to-prd 로 PRD를 만듭니다 ↳ 애매한 아이디어를 실행 가능한 상세한 스펙으로 바꾸는 단계입니다 Matt Pocock Skills: Thermo Nuclear Review: 8/ 에이전트용 작업 보드 만들기 (Run Codex with Kanban) ☑ agent 가 할 일을 여러 채널에 표시된 이슈 단위로 자동 관리 및 실행합니다 ↳ Symphony 처럼 issue 별 workspace 와 proof of work 를 남기는 흐름이 좋습니다 ↳ Backlog/In Progress/In Review/Done만 있어도 agent 관리가 쉬워집니다 Symphony: 9/ Codex 하네스에서 좋은 것을 가져오기 (Try and Trim Codex Harnesses) ☑ lazycodex, oh-my-codex 에서의 동작 원리를 이해합니다 ↳ 전부 사용하기보다 command, alias, workflow 원리를 보고 필요한 것만 남깁니다 ↳ 하네스는 설치보다 해체하면서 배우는 쪽이 더 오래 갑니다 10/ 쌓이는 로그와 트레이스 정리하기 (Clean Codex Traces) ☑ 오래된 trace, log, cache, temp output을 확인합니다 ↳ 이 작업에서 Mole 을 주로 활용합니다 ↳ 그럼에도 인증, 설정, MCP, skill, active session 은 함부로 지우지 않습니다 Mole: 11/ 초대 쿠폰으로 세션 한도 회복하기 (Reset Limits with Referrals) ☑ 아직 Codex를 안 써본 분에게 초대 링크를 보내고 reset coupon 을 확인합니다 ↳ 2026년 6월 24일까지 친구 초대 이벤트가 진행됩니다 ↳ 계정당 최대 5개의 리셋 쿠폰을 확보할 수 있습니다. Codex Download: 12/ Codex 를 멀티 모델 컨트롤 센터로 쓰기 (Connect Multiple Providers) ☑ Codex 를 main harness 로 두고 headless cli command 로 모델과 agent 를 목적에 맞는 subagent 로 붙입니다 ↳ 글 작성은 Claude ↳ 프론트와 3D 렌더링은 Antigravity ↳ 이미지, 영상, 콘텐츠 검색은 Grok Build ↳ 파일 검색과 repo 탐색은 Opencode (저렴한 AI) ↳ Codex는 이들을 지휘하는 컨트롤 센터가 됩니다 Custom Providers: —————

Codex Desktop App을 깔고 ChatGPT 플러스까지 구독했지만 여전히 기본 채팅만 치는 엔지니어가 많습니다. 도구를 단순히 쓰는 단계와 작업 시스템으로 엮어내는 단계는 다릅니다. 직접 쓰면서 정착시킨 12가지 워크플로 트릭을 정리했습니다.

기획과 실행의 망 분리#

ChatGPT는 기획을 맡기고 Codex는 실행만 전담하게 역할을 쪼갭니다. DevSpace 나 CodexPro 같은 MCP 서버를 로컬 저장소에 붙이는 방식으로 구현합니다. ChatGPT가 코드를 읽고 계획을 세우면 Codex가 로컬에서 그대로 실행하는 구조로 작업 생산성을 높입니다.

계정 한도나 작업 환경이 꼬이는 문제는 CLIProxyAPI 와 codex-auth 로 풉니다. 여러 계정을 API 게이트웨이 뒤로 묶어 라운드 로빈 방식으로 호출하면 한도 걱정이 사라집니다. 설정 파일을 분리해서 프로필을 전환하는 흐름도 매끄럽습니다. 인프라 비용을 줄입니다.

텍스트 백 줄보다 화살표 하나가 의도를 더 정확히 전달합니다. 인앱 브라우저로 Excalidraw 를 띄워 화이트보드에서 직접 그림을 그리며 수정 사항을 지시합니다. 프론트엔드 UI 밀도가 떨어질 때는 전용 디자인 플러그인을 붙여 완성도를 다듬습니다. 소통이 한결 수월합니다.

원격 전환과 에이전트 칸반#

퇴근길에 노트북을 닫기 전 원격 서버로 작업을 부드럽게 넘깁니다. 로컬 장비에서 시작한 무거운 코딩 작업을 클라우드 VPS나 사무실의 항상 켜진 PC로 토스합니다. 원격 연결 기능으로 끊김 없이 빌드와 테스트를 수행하며 개발 흐름을 이어갑니다.

에이전트가 처리할 일감은 Symphony 같은 도구로 이슈 단위로 관리합니다. 칸반 보드에 맞춰 에이전트가 알아서 PR을 올리고 테스트 결과와 코드 분석 리포트를 기록으로 남기게 만듭니다. 사람이 일일이 감시하지 않아도 작업이 굴러갑니다. 무척 편합니다.

무작정 구현에 들어가지 않고 grill-me 스킬을 써서 요구사항부터 날카롭게 고정합니다. 질문 공세를 받으며 애매한 아이디어를 구체적인 PRD 스펙 문서로 깎아내는 과정입니다. 뼈대를 세운 뒤에는 병렬 서브에이전트를 띄워 이미지나 리소스를 동시에 찍어냅니다. 속도가 빨라집니다.

멀티 모델 컨트롤 센터 구축#

lazycodex 같은 하네스는 무작정 설치하기보다 내부 동작 원리만 파악하고 걷어냅니다. 필요한 단축키와 워크플로만 남겨야 시스템이 가벼워집니다. 찌꺼기 로그와 캐시는 Mole CLI 도구로 주기적으로 밀어냅니다. 관리하기 쉽습니다.

Codex를 최상위 컨트롤 타워로 세우고 하위에 특화 모델을 서브에이전트로 붙여 지휘합니다. 글쓰기는 Claude에 맡기고 프론트엔드 렌더링은 Antigravity를 호출하며 로컬 파일 검색은 가벼운 오픈소스 모델을 씁니다. 각자 잘하는 영역에 배치해 효율을 크게 높입니다.

도구를 100% 활용하는 힘은 환경을 장악하는 커스텀 능력에서 나옵니다. 나만의 개발 공장을 짓는 재미가 쏠쏠합니다.

요약#

  • ChatGPT를 기획자로, Codex를 실행자로 분리하여 MCP 서버로 연결할 때 크레딧 효율과 개발 속도가 극대화됩니다.
  • 원격 서버 핸드오프와 Symphony 기반의 에이전트 칸반 보드를 결합하면 끊김 없는 자동화 파이프라인이 완성됩니다.
  • Codex를 최상위 컨트롤러로 두고 영역별 특화 모델을 서브에이전트로 배치하여 멀티 모델 오케스트레이션을 구축합니다.
