PPT 없이 마크다운 하나로 발표 자료를 끝내는 방법

6월 12, 2026 · 2 분 · Junho Lee
원문 manual
  • wikidocs.net/blog/@Allen/18956
목차

  • Git 버전 관리와 AI 협업
  • 세미나와 사내 공유 효율화
  • 요약

발표 자료를 준비할 때 내용 구상보다 레이아웃 배치와 폰트 크기 조정에 시간을 더 많이 쓰는 엔지니어가 많습니다. 정렬을 맞추고 색상을 고르는 과정은 디자인 비전공자에게 비효율의 극칩니다. 텍스트 작성에만 온전히 집중하고 디자인은 엔진에 맡기는 개발자 친화적인 방식을 고민했습니다.

디자인 대신 텍스트에 집중하기#

Marp 는 마크다운 문서 하나로 슬라이드를 완성하는 문서 변환 도구입니다. # 로 대제목을 쓰고 --- 로 슬라이드 페이지를 나누면 레이아웃과 정렬은 엔진이 알아서 처리합니다. 개발자가 PPT 상자 크기를 맞추려 마우스를 잡고 씨름할 필요가 없습니다. 시간 낭비가 없습니다.

텍스트만 빠르게 타이핑하면 되니 발표 자료를 만드는 속도가 기존 방식보다 훨씬 빨라집니다. 줄 바꿈이나 글머리 기호 설정 같은 자잘한 편집 작업은 도구에 완전히 위임합니다. 엔지니어는 오직 전달할 기술적 사실과 논리적 흐름에만 신경 씁니다. 생산성이 오릅니다.

Git 버전 관리와 AI 협업#

텍스트 파일 기반이라 Git 버전 관리를 자연스럽게 파이프라인에 녹입니다. 바이너리 형식인 PPTX 파일과 다르게 변경 사항을 라인 단위로 비교하고 동료 엔지니어와 코드 리뷰를 하듯 슬라이드를 고칩니다. AI 도구에 회의록이나 요구사항 명세서를 넘겨 마크다운 슬라이드 초안을 작성하라고 지시하는 작업도 매끄럽게 이어집니다. 궁합이 맞습니다.

동료와 협업할 때도 충돌 없이 병합하여 효율을 높입니다. 예전처럼 임시 파일명을 붙여가며 메신저로 주고받는 비효율이 사라집니다. 모든 변경 이력이 Git 커밋 로그로 투명하게 남아 추적하기 쉽습니다. 협업이 편합니다.

세미나와 사내 공유 효율화#

코드 리뷰나 사내 기술 세미나처럼 내용 전달이 핵심인 자리에서 Marp 는 뛰어난 효율을 발휘합니다. 작성한 마크다운 문서는 PDF, PPTX, HTML 등 필요한 포맷으로 즉시 변환하여 배포합니다. 한 번 구축한 커스텀 테마와 구조는 다음 발표에 그대로 복사해서 재사용하면 그만입니다. 불필요한 장식 요소를 걷어내니 오히려 메시지가 명확해집니다. 메시지가 삽니다.

복잡한 수학 공식이나 개발 소스 코드를 슬라이드에 삽입할 때도 마크다운 문법을 그대로 활용합니다. 구문 강조 기능이 기본으로 적용되어 소스 코드 가독성이 대단히 높습니다. 기술 정보를 정확하게 다뤄야 하는 엔지니어링 조직에 최적화된 환경을 제공합니다. 선택이 쉽습니다.

발표 자료 제작은 머릿속 생각을 타인에게 전달하는 과정입니다. 무거운 그래픽 툴 대신 가벼운 마크다운 파일 하나로 발표 준비를 마치는 경험은 꽤 유쾌합니다.

요약#

  • Marp를 사용하면 시각적 디자인 요소에 낭비되는 시간을 차단하고 오직 텍스트 콘텐츠에만 집중할 수 있습니다.
  • 마크다운 기반의 슬라이드는 Git 버전 관리와 라인 단위 변경 비교가 가능하여 협업 효율을 극대화합니다.
  • 텍스트 포맷 특성상 LLM API나 Claude Code 같은 AI 도구와의 연동 및 초안 작성이 매우 수월합니다.
