CHAPTER 02 : DataNexus

The problem of working with data
in natural language.

Enterprise data analytics always runs into the same wall. "Explore internal data through natural language" is easy to say — but in practice, table names look like T_CUST_MST, and "net revenue" carries different calculation logic across departments. LLMs cannot understand business context from DDL alone.

DataNexus tackles this by combining an ontology-native NL2SQL engine, GraphRAG, and a Data Catalog. An open-source composite architecture that handles both unstructured documents and structured databases together.

This blog documents the process — including the dead ends. Architecture decisions, reasons behind technology choices, failed experiments alongside successful ones.

// Three Pillars

01
Ontology-Native NL2SQL

Generates SQL from business semantics, not DDL. Represents cross-department definition differences clearly as a graph.

02
GraphRAG + Catalog

Connects unstructured documents and structured databases into a single graph — searchable, citable data assets.

03
Composable Architecture

Open-source composition avoids vendor lock-in. Each layer is independently replaceable.
